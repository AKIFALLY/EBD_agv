⚠️ 重要操作時機說明

  • 叫空車 → KUKA AGV 送空Rack到停車格

  • 派滿車 → 唯一的資料庫更新時機，將產品及Rack資訊存入系統
  • 註：只有在「派滿車」時，系統才會記錄產品和Rack的相關資訊

🔄 房間外作業工作流程

🏭 射出成型
眼鏡鏡片的初步成型製程，4台射出成型機進行原料熔融射出、模具成型冷卻、成品脫模檢查。
📱 OPUI 叫車
作業員透過 OPUI 平板進行叫車作業，設定產品類型和數量，選擇目標房間，叫空車到停車格。
🤖 KUKA AGV 搬運
KUKA Robot 400i 負責 Rack 的長距離搬運，包括空 Rack 配送到射出機停車格，滿載 Rack 搬運到房間門口。
🚛 Cargo AGV 銜接
Cargo Mover AGV 負責 Rack 與房間內製程的銜接，執行 Work ID 1001-1004，與房間內製程無縫銜接。
🏠 房間內製程
進入房間內進行精密處理，包括清洗、泡藥、預烘、烘烤等核心製程，完成產品的精密加工。
🚛 Cargo AGV 完成品取出
Cargo AGV 執行 Work ID 1011-1014，將處理完成的產品從出口傳送箱裝載到 Rack，準備送往包裝區。
📦 包裝出貨
完成品進入最終包裝流程，包括自動包裝作業、條碼標籤貼附、品質檢查、出貨準備和分類。

🎯 房間外三重系統協作

KUKA + Cargo + OPUI 整合：OPUI 負責人機協作和任務觸發，KUKA AGV 負責長距離 Rack 搬運和停車格配送，Cargo AGV 負責 Rack 與房間內製程的精密銜接，三系統無縫協作實現完整的房間外自動化流程

⏱️ 房間外典型作業時序流程

🏭 射出成型作業 (T+0分)
射出成型機完成眼鏡鏡片成型作業，Carrier 載具裝載完成。操作人員確認成型品質合格，準備進入 OPUI 叫車流程。
📱 OPUI 叫車作業 (T+2分)
操作員透過 OPUI 系統執行叫空車功能，系統自動分配 KUKA AGV 並建立運輸任務。KUKA Fleet API 接收任務請求，開始調度 Robot 400i。
🤖 KUKA AGV 空車配送 (T+5分)
KUKA Robot 400i 攜帶空架台前往射出區域。架台規格：S產品 16 Carrier/面，L產品 8 Carrier/面，A/B雙面設計。AGV 精確定位至指定工位。
📦 架台載料作業 (T+10分)
操作員將成型完成的 Carrier 載具依序載入架台。A面優先裝載，滿載後裝載B面。注意：此階段為純人工作業，系統不會即時更新。
🚀 OPUI 派滿車確認 (T+15分)
載料完成後，操作員透過 OPUI 執行派滿車功能。此時系統才將產品及Rack相關資訊存入資料庫，並生成運輸任務，KUKA AGV 準備前往房間入口。
🤖 KUKA AGV 滿車運輸 (T+20分)
KUKA Robot 400i 攜帶滿載架台，經由 KUKA Fleet Manager 路徑規劃，精確導航至房間入口，將架台放置在指定位置後離開。
🔄 Cargo AGV 架台接收 (T+25分)
資料庫依據 KUKA AGV 搬運任務完成後更新 Rack 的所在地，系統依據 Rack 在房間入口位置產生任務給 Cargo AGV，執行 Work ID 1001-1004 進行架台處理作業。
註：KUKA AGV 與 Cargo AGV 不直接對接，透過資料庫狀態同步實現協作。
🏠 房間內製程交接 (T+30分)
Cargo AGV 透過機械臂從架台上取下 Carrier 載具，轉移至房間入口傳送箱。房間內 Loader AGV 接手進行清洗、泡藥、預烘、烘烤等製程。
⏳ 房間內製程處理期間 (T+30分 - T+70分)
房間內製程進行期間，KUKA AGV 可處理其他射出區域的運輸任務。Cargo AGV 準備空架台，等待房間內製程完成。
🚛 Cargo AGV 完成品架台裝載 (T+70分)
房間內製程完成，Cargo AGV 執行 Work ID 1011-1014，將處理完成的 Carrier 從出口傳送箱裝載至架台，準備送往房間外。
🤖 KUKA AGV 完成品運輸 (T+75分)
KUKA Robot 400i 在房間出口接收滿載完成品的架台，透過 Fleet API 路徑最佳化，運送至人工拆片區域。
👥 人工拆片作業完成 (T+80分)
架台送達人工拆片區域，作業員將Rack上的Carrier拿下並取出眼鏡。空Rack回收移至對應回收區域，完成品進入包裝、品檢、出貨流程。整個房間外製程循環完成。
註：OPUI「確認送達」是在叫空車時，確認空Rack送達停車格後的操作，與此階段無關。
