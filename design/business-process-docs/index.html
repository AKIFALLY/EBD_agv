🎯 入門必知：RosAGV 系統架構概念

RosAGV 是一套眼鏡生產工廠的智能製造執行系統，整合了多套AGV系統協同作業：

  • CT AGV (3車型)：系統直接管理的核心車隊 - Cargo搬運架台、Loader精密載料、Unloader後段卸料
  • KUKA AGV (2台)：外部整合的Robot 400i，負責房間外走廊搬運和架台轉向
  • 95+ Work ID：涵蓋房間內精密製程(25+)、房間外物流作業(30+)、系統整合任務(40+)
  • 雙環境架構：車載即時控制 + 中央車隊管理，透過容器化部署和Zenoh通訊

🔧 技術支援資訊

⚠️ 重要概念說明：

  • 雙環境架構：AGV車載系統(即時控制) + AGVC管理系統(車隊調度)
  • CT vs KUKA AGV：CT AGV系統直管3車型，KUKA AGV透過API整合
  • Work ID分層：95+任務編碼涵蓋房間內外製程、系統整合、設備控制
  • 容器化開發：所有ROS 2程式須在Docker容器內執行和測試
  • 通訊機制：Zenoh RMW跨容器 + KUKA Fleet API + PLC協議

🚀 快速上手指令

🔍 系統診斷 (宿主機執行)

# 快速健康檢查
r agvc-check # AGVC 管理系統健康檢查
r containers-status # 所有容器運行狀態
r network-check # 網路連接和端口檢查
r quick-diag # 快速綜合診斷

🐳 容器開發 (進入開發環境)

# 進入 AGVC 管理容器
agvc_enter # 自動載入環境
all_source # 載入所有工作空間
# 進入 AGV 車載容器
agv_enter # 進入車載開發環境

🏗️ 常用開發指令 (容器內執行)

# ROS 2 開發
colcon build --packages-select 套件名
ros2 launch 套件名 launch檔案.py
ros2 topic list # 查看主題列表
# 系統狀態檢查
check_system_status # 容器內系統狀態
check_zenoh_status # Zenoh 通訊狀態
